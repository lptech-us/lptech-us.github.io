About LP Code

On November 22, 2003, version 1.0 of LP Code passed the national identification under the auspices of the Chinese Automated Identification Technology Association and the highly respected seniors in the computer field, He Dequan and Ni Guangnan.

For more than a decade, I have been combining the latest requirements of the Internet of Things and smart cities, and insisted on the continuous improvement and improvement of LP Code, making LP Code the most advanced two-dimensional barcode in the world. It provides a solid foundation for China to fight for a voice on the international Internet of Things identification system.

About LP Code Versions

2003: LP Code Version 1.0

LP Code version 1.0 is a bi-directionally readable 2D barcode with comprehensive information synchronization

LP code version 1.0 is composed of a set of rectangular elements of the same size with different attributes. These rectangular elements are called information units. Attributes are usually black and white. An information synchronization unit is a group of special information units. They usually consist of a black and white information unit. Rombay code provides a full range of information synchronization control characters and overall reading direction control characters. It can provide effective information to the anti-distortion algorithm and guide the overall reading direction of the barcode (horizontal and vertical directions of the barcode) to realize software bidirectional reading.

2013: LP Code Version 2.0

LP Code version 2.0 has comprehensive information synchronization and supports icons

LP Code version 1.0 has a weaker positioning performance because the guide bar code is slightly longer than the synchronous bar code. Because any image can have interference, also called noise. There are many types of image noise, with point noise being the most common. The probability of point noise falling next to the synchronous bar code is very high. When there is a little noise next to the synchronous barcode, it is often difficult for the image processing system to distinguish whether it is a synchronous barcode or a guide barcode. This is also an important reason for mistranslation of the LP Code. And because the LP Code version 2.0 supports the icon function, the bidirectional reading function can no longer be adapted, so the structure of the lobster code must be modified.

In LP Code version 2.0, a bar-shaped bar code that is next to the bar code and is the same height as the bar code is used as the positioning bar code. Since straight long bar noises in the noise are rare, the probability of a straight bar noise next to the synchronous barcode and equal to the bar code is almost zero, so we believe that the signal strength of the new version of the barcode positioning system is much greater than that of LP Code. At the same time, the left and right sides are asymmetric, and the head signal and the tail signal can be omitted, which simplifies the error correction coding of the format information. From the perspective of design, version 2.0 is also more beautiful than version 1.0.

2016: LP Code Version 3.0

LP Code version 3.0 has improved icon support

LP Code version 3.0 is the most perfect two-dimensional code in the contemporary era. It retains all the advantages of LP Code 1.0 and 2.0. Its most amazing feature is that it does not require any position detection graphics. It can be done with fast and reliable software algorithms. Achieve omnidirectional reading. It is another leap in the history of bar code based on the 1.0 version of the two-way reading technology, and it is a new milestone in the development of bar code technology.
